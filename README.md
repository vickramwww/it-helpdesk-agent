# The Final Challenge — Competition Brief

## Format
- **Teams:** 5-8 people
- **Time:** 50 minutes of active work
- **Tool:** Claude Code (terminal or VS Code)
- **Deliverable:** Zip file with 3 files

## Rules
1. Pick ONE scenario. Don't switch.
2. Claude Code is your only coding tool. No copy-pasting from external sources.
3. Start by creating a `CLAUDE.md` — this is context engineering in action.
4. Depth beats breadth. 3 challenges done well > 5 done poorly.
5. When time is called, zip your folder and email it to the proctor.

## Scenarios

| # | Scenario | Best For |
|---|----------|----------|
| 1 | Code Modernization | Developers, backend engineers |
| 2 | Data & Analytics | Data analysts, BI engineers |
| 3 | Agentic Solution | AI enthusiasts |

See the individual scenario files for details and challenges.

## Submission (exactly 3 files)

1. **README.md** — Use the template in `submission-template/`
2. **CLAUDE.md** — The one you used during the competition
3. **presentation.html** — Self-contained HTML summary built with Claude Code

## Judging Categories
- Most Production-Ready
- Best Architecture
- Best Claude Code Usage
- Most Complete
- Best Presentation

Claude will be the judge.
